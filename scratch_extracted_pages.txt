3.1.3 Training prediction models. Using the sampled graphs and their graph-based features, we
train prediction models to estimate GRL accuracy for a downstream task after community-based
partitioning. We also train a model to predict the accuracy when we do not apply graph partitioning
(i.e., no-partitioning strategy). In this study, we select five representative community detection
algorithms (Louvain [2], Label propagation [33], Fast greedy [6], Infomap [38], and Leiden [48])
considering both their effectiveness and computational costs while excluding inefficient algorithms
(e.g., Spinglass [35] and Edge betweenness [11]) based on the investigations of various community
detection algorithms [8, 9, 56].
Our method aims to predict GRL accuracy of a specific downstream task when partitioning the
graph using community detection algorithms. Specifically, for each sampled graph, we partition the
graph using the five community detection algorithms and obtain embeddings on the partitioned
subgraphs using a base GRL model. We then apply the embeddings learned from GRL to specific
downstream tasks, such as link prediction and node classification, and then measure the accuracy
of learned representations for each downstream task. We employ ML-based regression models as
prediction models because they offer greater efficiency and flexibility while providing a moderate
level of learning accuracy. Then, we use the model with the best prediction performance for each
case.
The offline phase involves 1) sampling graphs, 2) measuring the representation accuracy, including
running the GRL model for each sampled graph with various community detection algorithms,
and 3) building the prediction model for each community detection algorithm. The extensive
considerations for diverse community detection algorithms require a massive amount of time, but
notably, these do not affect online running time. We show the details of the required times for the
offline and online phases in Section 4.4.1 and Section 4.4.2, respectively.

4.3.5 Effects on multi-level partitioning of ML-GRL. To assess the impact of multi-level partitioning

algorithm selected in level 1 (simply, Fixed-2L), 2) three-level GRL fixing with a community detection
algorithm selected in level 1 (simply, Fixed-3L), and 3) three-level GRL while choosing community
detection algorithms according to our prediction models (simply, Dynamic-3L). Here, the selected
community detection algorithms vary according to the used dataset and the GRL model for each
method. We compare our final ML-GRL (i.e., without level restriction) with these variations.
Tables 5 and 6 show the link prediction and node classification results, respectively. When
comparing the three variations, Fixed-2L clearly improves the accuracy of the original models
shown in Tables 3 and 4. Fixed-3L further enhances Fixed-2L, showing the effects of a deeper
partitioning level. Dynamic-3L improves Fixed-3L, indicating the effectiveness of our predictive
partitioning. Finally, ML-GRL consistently improves the performance of all the variations, including
Dynamic-3L, showing the effectiveness of the adaptive extension of community-based partitioning
to deeper levels.

B. Performance on Link Prediction
Link prediction aims to predict the presence or absence of
an edge between two nodes. With the node embeddings learned
through GRL, we build a logistic regression model for random
walk-based models or attach a fully connected layer to the last
layer of GNN-based GRL models to predict the probability
of edge existence for a given node pair (i.e., adopting diverse
prediction models). For the learning phase, we create a training
set by randomly eliminating 10% of the edges from the graph.
Subsequently, the GRL models are trained using the remaining
graph. The test set comprises the 10% removed edges as positive
samples, along with an equal number of unconnected node pairs
serving as negative samples.
Table VI summarizes link prediction results for eight base
GRL models on the Foursquare, CiteSeer, and WikiCS datasets.
The results demonstrate that our model not only can be equipped
with diverse GRL models but also improves the representation
accuracy while signiﬁcantly reducing the learning time for each
evaluated base GRL model. Speciﬁcally, our model enhances
accuracy by a range of 1.35% to 12.47% and reduces learning
time by 31.79% to 48.1% across various base models and
datasets. Notably, the accuracy improvement achieved by our
model with AnECI is comparatively modest compared to other
base models. We assert that this is due to the fact that AnECI’s
embeddings already effectively retain community information
acquired through GRL. Nevertheless, we note that, while our
model shows a modest improvement in accuracy, it signiﬁcantly
reduces the learning time by 38.2%.
